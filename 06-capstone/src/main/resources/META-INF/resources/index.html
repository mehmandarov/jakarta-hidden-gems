ConfSpeakerHub phase 1 — gems #1–5

Personal radar for tech conferences with grounded LLM matchmaking.

Gem #1 (Jakarta Interceptors) traces every endpoint into Jaeger. Gem #2 (Jakarta Authorization) filters the conference list per logged-in speaker from a hot-reloadable JSON policy file. Gem #3 (Jakarta Connectors + JPA) ingests CFP webhooks onto a JMS queue, dedupes + persists them in one JTA tx, and audits speaker actions to a GitHub gist. Gem #4 (Jakarta REST SSE + @Asynchronous) pushes new rows live to every browser tab without a single line of WebSocket code. Gem #5 (Jakarta JSON-P) streams a local LLM's analysis of any event — TL;DR, tags, and three of your own talks that fit it — token-by-token, through one JsonParser.

What's live right now

  • GET /api/health — liveness probe (public)
  • GET /api/events — conference list, filtered per speaker, @Traced @Timed
  • GET /api/events/{id} — single event lookup
  • GET /api/events/stream — SSE snapshot + live updates (gem #4)
  • GET /api/events/{id}/analyze — SSE live LLM matchmaking (gem #5)
  • POST /api/ingest — webhook landing site → confs.events.incoming
  • POST /api/events/{id}/action — SAVE / DISMISS / APPLY (DB write + audit publish in one JTA tx)
  • BASIC auth: rustam:hiddengems (Java/cloud-native/AI/devex, en/nb) or maria:flutter (mobile/AI, en/es)
